# Security policy

Do not report vulnerabilities through public issues. Use the repository's
private security advisory channel and include affected versions, impact,
reproduction steps, and any proposed mitigation.

Never include credentials, session cookies, private account data, or active
tokens in a report or test fixture. Revoke exposed credentials before reporting
them.

Pre-release versions receive security fixes on a best-effort basis. A supported
version policy will be published with the first stable 2.0 release.
